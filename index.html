MELD — Class Imbalance
Dominant: Neutral — 47%
Rare: Fear — 3%, Disgust — 3%
FER-2013 — Class Imbalance
Dominant: Happy — 25%
Rare: Disgust — 2%
Because of this imbalance, text and fusion training used class weights, and we evaluated everything with weighted F1 rather than relying on accuracy alone.
⚠ Important design note: Text and image were paired by the same label, not taken from the same original scene. Label-aligned pairs ≠ real synchronized text + face pairs.

Happy — all four models predicted correctly. This is a clean case, and happy is also one of the more common and visually clearer classes.

Challenges · limitations · improvements
What this setup can and cannot claim
01Label-aligned, not truly paired

Text and images were matched by emotion label, not from the same MELD video moment.

02Class imbalance

Fear and disgust remained difficult because they had fewer examples in both datasets.

03Transfer learning underperformed

MobileNetV2 struggled because ImageNet color features did not transfer well to 48×48 grayscale faces.

04Audio was not included

The project used text + image only; tone, pitch, and pacing are future work.

05Training time / compute constraints

RoBERTa context experiments, Optuna trials, CNN training, and fusion training required significant Colab GPU time.

What worked

  • Context-aware RoBERTa text model
  • Custom CNN performed better than MobileNetV2
  • Probability-level fusion improved performance
  • MLP fusion gave the best weighted F1 / lowest loss
  • Gradio demo made the system testable

What must improve / future work

  • Use real MELD video frames instead of FER-2013 substitute images
  • Add audio to build text + image + audio emotion detection
  • Use embedding-level fusion instead of only probability-level fusion
  • Improve fear/disgust using oversampling, focal loss, or more data
  • Try fine-tuning MobileNetV2 instead of keeping it fully frozen
